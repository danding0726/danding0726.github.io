## Blog cosmos v2 direction

- The homepage should feel like entering a universe, not operating a website dashboard.
- Removed the left navigation and command-style side rail; all primary exploration starts from planets.
- Planets need distinct visual identities (origin/organic, crystalline compute, gas relay) while sharing one orbital design system.
- English is appropriate for short system labels and atmospheric copy; Chinese remains useful for explanations and article content.
- Navigation model: galaxy overview → focus a planet → select an article station → read the full article → return to that planet.

## Dead-star expedition worldbuilding

- The whole blog now uses a post-collapse expedition setting: the visitor is a recovery astronaut, while Danding is a pathfinder who arrived earlier and left recoverable traces behind.
- Human-authored technical posts are `PATHFINDER TRACE` field logs. AI daily/insight content is explicitly separated as `MACHINE ECHO` with no human signature.
- Planet identities are lore-first but retain technical domains: FIRST CAMP / identity, GLASS WASTE / model quantization, ASHEN RELAY / inference serving.
- Dark mode is `NIGHT VISOR`; light mode is a low-saturation `PAPER LOG` field-manual view.
- Production Jekyll excludes private workspace files (`AGENTS.md`, memory files, identity/user/tool instructions, reports) so they cannot be emitted as site pages.
- Visual-density refinement: keep the dead-star premise and differentiated planets, but avoid turning the blog into a full HUD. Favor editorial whitespace, serif titles and readable article structure; use mission-system details sparingly.
- Navigation preference: primary destinations must be immediately recognizable as clickable blog links. Use familiar icons and Chinese action labels; keep fictional English terminology secondary rather than relying on abstract device symbols.
